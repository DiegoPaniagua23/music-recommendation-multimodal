En este markdown vamos a definir cómo se descargaron los datos para el Proyecto Final de la materia MIR. Primero que nada, vamos a definir los datos.

* Musicales (Audio): 10K canciones con duración de 30s en archivos WAV o MP3, obtenidas de YT-DLP. Se realizó un recorte temporal (windowing), cortamos el segmento del segundo 00:30 al 01:00, Remuestramos el audio de 44.1Hz a 22,050 Hz. Esto debido a que seguimos el Teorema de Nysquits (importante mencionar y desarrollar el teorema para que se entienda), 22kHz es suficiente para representar frecuencias de hasta 11kHz donde reside la mayor parte de la información que distingue un género musical. Esto también reduce el tamaño del archivo a la mitad sin perder datos útiles para el modelo. También mezclamos los canales estéreo en un solo canal mono porque la información espacial es irrelevant para clasificar un género, pero duplica la complejidad computacional. Al usar mono, simplificamos la entrada del modelo.  Para minar estos datos se creó un script que toma los metadatos limpios y genera una búsqueda optimizada para YouTube. Se utilizó yt-dlp para buscar al menjor candidatos. Configuramos el script para preferir videos etiquetas como "Audio" o de alta calidad (bestaudio). Para este proceso, recurrimos a VPNs gratuiras pues YouTube bloquea las descargas si detecta actividad extraña o robotizada de descargas.
* Texto: lyrics correspondientes de las 10K canciones, obtenidas de Genius a través de su API para Python. Se hizo una búsqueda en cascada para ir encontrando las lyrics, obtuvimos aproximadamente 8.5K canciones con letra, y el resto sin letra, correspondiendo sobre todo a canciones instrumentales con compositores como Hans Zimmer o Frank Zappa. Es decir, las canciones sin letra suelen corresponder a música instrumental, orquestas, clásica o soundtracks de películas.
* Imagenes: a través de los tracks ID del dataset de Spotify en Kaggle. El catálogo de las canciones es completamente integrable con la API de Spotify para Python, llamado SpotiPy. Al solicitar una canción a la PAI de Spotify, esta nos regresa infromación del álbum con la carátulas en tres maños diferentes: 30x30, 300x300 y 600x600. Para este proyecto nos quedamos con el tamaño de 300x300.
* Features: a través del dataset de SPotify en Kaggle. Pudimos encontrar cosas como qué tan bailable es la canción, el mood, e incluso el género.
